## 📖 产品简介

音频加密解密工具是一款基于HTML5技术开发的Web应用，用于加密和解密音频文件，支持多种加密格式，提供自动后台解密播放功能。

该工具采用先进的AES-GCM 256位加密算法，确保您的音频文件安全可靠。同时，提供了现代化的用户界面和流畅的用户体验，支持基础模式和高级模式切换，满足不同用户的需求。

## 📋 支持格式

## 🚀 使用方法

### 1. 基础使用流程

#### 1.1 打开工具
- 直接在浏览器中打开 `audio_encryptor.html` 文件
- 系统将自动生成设备码和5个密钥（存储在浏览器本地）

#### 1.2 密钥管理
- **查看密钥**：点击播放器右上角的⚙️图标打开设置面板
- **生成新设备码**：在设置面板中，在设备码输入框中输入自定义设备码，或点击「生成设备码」按钮
- **生成新密钥**：点击「生成密钥」按钮，系统将基于当前设备码生成新密钥
- **重新生成5个密钥**：点击「重新生成5个密钥」按钮，可以批量生成5个新密钥
- **使用指定密钥**：在密钥列表中点击「使用」按钮，可以切换到指定密钥
- **自主添加密钥**：在密钥输入框中输入32位十六进制密钥，点击「添加密钥」按钮
- **删除密钥**：在密钥列表中点击「删除」按钮，可以删除指定密钥

#### 1.3 音频加密保护功能
- 在设置面板中，可以找到「音频加密保护」开关
- 开启时：可以进行音频加密和解密操作
- 关闭时：无法进行加密和解密操作，保护音频文件安全

### 2. 音频文件处理

#### 2.1 选择音频文件
- 单文件选择：点击「选择音频文件」按钮，选择一个或多个音频文件
- 文件夹选择：点击「或选择文件夹」按钮，选择包含音频文件的文件夹
- 文件拖放：直接将音频文件拖放到页面上

#### 2.2 音频加密
1. 确保已生成密钥且「音频加密保护」功能已开启
2. 选择要加密的音频文件（MP3、FLAC、WAV格式）
3. 点击「加密」按钮，系统将开始加密
4. 加密完成后，处理后的文件将显示在「处理后的文件」列表中
5. 可以点击「下载」按钮保存加密文件，或点击「播放」按钮直接播放

#### 2.3 音频解密
1. 确保已生成密钥且「音频加密保护」功能已开启
2. 选择要解密的加密音频文件（SMP3、SFLAC、LGG、LAV格式）
3. 点击「解密」按钮，系统将开始解密
4. 解密完成后，处理后的文件将显示在「处理后的文件」列表中
5. 可以点击「下载」按钮保存解密文件，或点击「播放」按钮直接播放

### 3. 高级音乐播放器使用

#### 3.1 播放控制
- **加载文件**：点击📁按钮加载音频文件
- **播放/暂停**：点击▶️/⏸️按钮控制播放状态
- **快进/快退**：点击⏩/⏪按钮可以快进/快退10秒
- **进度调节**：点击进度条可以跳转到指定播放位置
- **音量调节**：点击音量条可以调节音量大小

#### 3.2 一键操作
- **一键加密**：点击「🔒 一键加密」按钮，可以加密当前播放的音频
- **一键解密**：点击「🔓 一键解密」按钮，可以解密当前播放的加密音频
- **一键扫描导入**：点击「📁 一键扫描导入」按钮，可以快速导入音频文件

#### 3.3 歌曲库管理
- **查看歌曲列表**：在「歌曲列表」部分可以查看已导入的歌曲
- **播放全部**：点击「播放全部」按钮，可以连续播放所有歌曲
- **清空列表**：点击「清空列表」按钮，可以清空歌曲列表
- **播放指定歌曲**：点击歌曲旁边的「播放」按钮，可以播放指定歌曲
- **下一首播放**：点击「下一首」按钮，可以将指定歌曲设置为下一首播放
- **删除歌曲**：点击「删除」按钮，可以从列表中删除指定歌曲

### 4. 自动后台解密播放

1. 确保已生成密钥
2. 系统默认始终开启自动解密功能，无需手动操作
3. 直接播放加密音频文件，系统将自动检测并在后台解密
4. 可以通过拖放加密音频文件到页面，或从歌曲库中选择加密歌曲
5. 解密状态将显示在播放器底部的「解密状态」区域
6. 支持多文件自动排队解密，提供无缝播放体验

## 📝 注意事项

1. 密钥是基于设备码生成的，同一设备码生成的密钥相同
2. 请妥善保管设备码和密钥，丢失将无法解密文件
3. 加密后的文件无法直接播放，需要使用本工具解密或开启自动解密功能
4. 自动解密功能需要先生成密钥才能使用
5. 建议在使用前备份原始音频文件
6. 本工具仅用于学习和研究目的，请勿用于非法用途
7. 使用本工具加密的文件仍可能存在被破解的风险，请根据实际需求选择合适的加密强度和保护措施
